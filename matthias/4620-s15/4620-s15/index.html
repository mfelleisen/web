Building Extensible Systems

Welcome to Building Extensible Systems.

Here is the official abstract:

Deals with the design of extensible software systems. An extensible system enables clients to add functionality both statically as well as dynamically. Examples of such systems are operating systems, game servers, and web browsers. – Describes the classic systems built on C-like languages with unsafe, manual memory control and the more recent systems built on Java-like languages with safe, automated memory management. – Then introduces the Rust programming language, which combines the efficiency of C with safe manual memory control via type specifications and compiler constraints. Students will build systems using all three settings but will focus on the Rust approach. Students will evaluate their work via essays and memos.

And this is what we original proposed:

Extensible systems enable programmers to dynamically add components to a running instance. Conventional operating systems, game servers, and web browsers are examples of such systems.

In the past software engineers have built extensible systems in either a C-like language with unsafe, manual control over memory or in a Java-like language with safe, automated control over memory. Over the last four years Mozilla has designed and built the Rust programming language with the goal of combining the advantages of both approaches. Rust’s interface language allows programmers to control memory manually via type specifications and in a safe manner.

This course will teach students how to build systems in all three settings with a focus on the Rust model of programming. Students are expected to know some C/C++, Java, and remember their freshman year.In addition to systems building, the course will involve a writing component.
