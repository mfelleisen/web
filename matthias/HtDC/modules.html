1 Modules

Programs tend to grow so large that it is impossible for one person to create all the code. Instead, many programmers—sometimes thousands of them—create pieces of the program and later integrate them into one large system. In general, computer scientists refer to these pieces as modules but different programming languages have different names for them. Java, for example, uses the word package.

The key purpose of a module is to provide a programmer with private space. In this private space, the programmer can create code and a test suites and often other documents without paying attention to what other programmers do. Most importantly, if two different programmers wish to use the name myFavoriteFunction they can do so without interfering with each other—as long as these definitions are placed in a module.

Of course, the purpose of creating many modules is to integrate them later into one program. In order to integrate them, the creators of modules must “publish” the names of some of their functions so that other programmers can use them in their modules. These publicly visible names form the interface of a module. Everything else in the module—the code for the public names, other functions—is “private” and, as such, of no interest to programmers other than the creator of the module. As a matter of fact, the programmer is free to change these private parts as long as the public names continue to mean what other programmers were told about them.

1.1 Modules in PLT

In PLT, a plain module is a container for all kinds of definitions and comments: structures, data, functions, contracts, and purpose statements. For now, you may create a module with DrScheme’s Module language. When you choose this language, DrScheme inserts a line for you:

#lang scheme

The first word in this #lang tells DrScheme that this piece of code is a module, and the second word scheme specifies in which programming language you are going to write this module.

#lang scheme
(define (f2c f)
  (aux f))
 
; Number -> Number
; the workhorse for f2c
(define (aux f)
  (* 5/9 (- f 32)))
 
; ——————————————————————
 
(provide
  f  ; Number -> Number
  ; convert a Fahrenheit temperature to a Celsius temperature )
