Every team in CS4500 has a row in the table. To find your row, check your email inbox for a message with the subject:

cs4500 team identification number

and use the 4-digit PIN in that message to locate your row.

DO NOT SHARE your 4-digit PIN with another team.

The MF column shows your score on our test suite.

  • 0 = improperly placed/named or non-executable file

  • 6 = fails all tests

  • 16 = max score

The Code Inspection column shows your code inspection score.

  • 20 = max score

The Test Bonus column shows extra credit points.

  • 1 = submitted 1 valid test that caused at least one student executable to fail

  • 5 = max score

Click any cell for details.

13.1 Test Fest

Each row represents a valid xrules file.

Each column represents a suite of valid JSON tests.

Each cell contains a 5-character string representing the results of running the tests on the executable. There are 3 possibilities for the character in the i-th position of each string:

  • x = the i-th JSON test was invalid, or not submitted

  • 0 = the executable passed the i-th test

  • 1 = the executable failed the i-th test

Interpretation:

  • To see how many student tests your xrules failed, find the row labeled with your team PIN and count the number of 1 characters.

  • To calculate the bonus points from your tests, find the column labeled with your team PIN and count the number of unique positions in the cells that contain a 1 character.
