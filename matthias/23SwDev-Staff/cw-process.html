4 Code Walk: Process

Purpose A section TA assists the instructor with conducting code walks: taking notes, managing questions from on-line participants, observing the presenters and the panelists, making suggestions, and correcting the instructor when needed.

Selection and Invitation Michael has allocated a staff-only git repository for code-walk grades and feedback:

    github.khoury.northeastern.edu/CS4500-F23/code-walk/

Before every lecture, Michael will place forms into four sub-directories of this repository (one per section) and thus assign roles among students. For example, he may create

    Michael/

        MF-915/

            9-28/

                 1-presenter1.txt

                 1-presenter2.txt

                 1-head.txt

                 1-assistant.txt

                 1-secretary.txt

    

                 2-presenter1.txt

                 2-presenter2.txt

                 2-head.txt

                 2-assistant.txt

                 2-secretary.txt

    

        MF-1030/ ...

    

        BL-1030/ ...

    

        BL-0135/ ...

This are verbatim file names. The names of the presenters and panelists are in the files. The form files will have this shape:

    Name: John Doe

    Email: doe.j@northeastern.edu

    Grade: ok+ | ok | ok- | 0

    

    ... informal remarks ...

Michael will send an email with the same information to the course mailing list. The lead instructor will post the information on the course site.

Michael will also send email invitations to the chosen presenters and panelists on the morning of the presentation day (or late night before the day). CC the instructor of the section. — The goal is that students don’t have time to prepare, but do have time to opt-out if there is an emergency.

Results The section TAs will commit grades and comments via these files to this repository on per code-walk basis.

The section TAs will send out grades for the presenters and panelists, after the postmortem meeting.

The instructors will send grades for the secretary memos after evaluating them; they will CC Michael for record keeping in the respective files.

See Code Walk: Post Mortem for details on the evaluation session.
