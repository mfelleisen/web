6 Code Walk Process Draft

Ben has allocated a staff-only git repository for code-walk grades and feedback:

github.ccs.neu.edu/CS4500-F20/code-walk/

Before every lecture, Ben will place form files into sub-directories of this repo and thus assign roles among students. For example,

    

    9-18/

        pair-student1.txt

        pair-student1.txt

        pair-head.txt

        pair-reader.txt

        pair-secretary.txt

Since pairs are (mostly) section based, there should be no confusion as to when someone presents.

Ben will send an email with the same information to the section head (with CC to the instructor).

The job of the section TA is to

  1. Review the presentation assignment for her section. Speak up — or edit — if there is a conflict.

  2. Shortly before class, email the presenters and panelists:

    • Suzanne: try to email late (after 10pm) on the night before class

    • Julia and Kanika: try to email in the morning (6am to 11am) on the day of.

    The goal is that students don’t have time to prepare, but do have time to opt-out if there is an emergency — we will pick replacements.

  3. After class, commit grades and comments to this repository. The form files will have this shape:

    Here is the format for a report:

        Name: John Doe

        Email: doe.j@northeastern.edu

        Grade: ok+ | ok | ok- | 0

        

        ... informal prose ...

Ben will collect the comments and email them to the students, with a cc to the section head.
